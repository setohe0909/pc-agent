                Flujo Discord
                
Discord envia solicitudes al bot. Las decisiones de trading pasan por assistant-runtime solo si fueron aprobadas en el canal autorizado.

              

              

                Flujo conocimiento
                
ingestion-worker lee fuentes, usa Ollama para embeddings y guarda chunks en Supabase/pgvector.

              

              

                Flujo control
                
La UI habla con control-api para estado, configuracion runtime, fuentes y verificaciones.

              

            

          

        


        

          

            

              
Wiki de uso

              
Guia rapida para operar PC Agent sin perderse entre servicios.

            

          


          

            

              
1. Arranque

              
Levanta los servicios base con Docker y abre la consola en localhost:8080.

              docker compose up --build
            


            

              
2. Configuracion

              
En la pestaña Configuracion define URLs de assistant-runtime, MentisDB, Langfuse, Ollama y Supabase.

              http://localhost:8100 · http://localhost:9471 · http://localhost:3000
            


            

              
3. Discord

              
En Discord configura token, canales y aprobadores. Las decisiones de trading solo pasan por Discord.

              !ask · !research · !approve_trade
            


            

              
4. Conocimiento

              
Agrega fuentes RSS o website. El worker descarga contenido, genera embeddings con Ollama y guarda en Supabase.

              mxbai-embed-large · 1024 dimensiones
            


            

              
5. Supabase

              
La publishable key permite lectura publica. La escritura de embeddings requiere service role en backend.

              knowledge_sources · knowledge_documents · match_knowledge_documents
            


            

              
6. Seguridad de trading

              
El runtime rechaza trade_decision y open_position si no vienen del canal Discord autorizado con aprobacion.

              DISCORD_REQUESTS_CHANNEL_ID · DISCORD_APPROVER_USER_IDS
            


            

              
7. Observabilidad

              
Langfuse corre con el perfil observability. Usalo para inspeccionar conversaciones y llamadas a herramientas.

              docker compose --profile observability up --build
            


            

              
8. Estado actual

              
PC Agent v0.2.0 tiene consola, configuracion runtime, Discord gate, Supabase pgvector e ingestion base. Kalshi live sigue protegido pendiente de adapter real.
